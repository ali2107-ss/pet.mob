# 🚨 СРОЧНО: Исправление ошибки заказов

## Проблема
При оформлении заказа выходит ошибка, потому что таблица `order_items` в Supabase не имеет нужных колонок.

## Решение (5 минут)

### Шаг 1: Откройте Supabase
1. Зайдите на https://supabase.com
2. Откройте ваш проект
3. Перейдите в **SQL Editor** (слева в меню)

### Шаг 2: Выполните SQL скрипт
Скопируйте и вставьте этот код в SQL Editor и нажмите **RUN**:

```sql
-- Удалить старую таблицу
DROP TABLE IF EXISTS order_items CASCADE;

-- Создать правильную таблицу
CREATE TABLE order_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id UUID NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  product_id TEXT NOT NULL,
  product_name TEXT NOT NULL,
  partner_id UUID REFERENCES auth.users(id) ON DELETE SET NULL,
  quantity INTEGER NOT NULL,
  price_at_purchase NUMERIC(10, 2) NOT NULL,
  total NUMERIC(10, 2) NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Индексы
CREATE INDEX idx_order_items_order_id ON order_items(order_id);
CREATE INDEX idx_order_items_product_id ON order_items(product_id);
CREATE INDEX idx_order_items_partner_id ON order_items(partner_id);

-- Включить RLS
ALTER TABLE order_items ENABLE ROW LEVEL SECURITY;

-- Политики безопасности
CREATE POLICY "order_items_select" ON order_items
  FOR SELECT USING (
    EXISTS(
      SELECT 1 FROM orders WHERE orders.id = order_items.order_id AND orders.user_id = auth.uid()
    )
    OR auth.uid() = partner_id
  );

CREATE POLICY "order_items_insert" ON order_items
  FOR INSERT WITH CHECK (
    EXISTS(
      SELECT 1 FROM orders WHERE orders.id = order_items.order_id AND orders.user_id = auth.uid()
    )
  );
```

### Шаг 3: Проверьте таблицу orders
Выполните этот код для проверки (если orders не существует, он создаст её):

```sql
CREATE TABLE IF NOT EXISTS orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  total_amount NUMERIC(10, 2) NOT NULL,
  status TEXT NOT NULL DEFAULT 'pending',
  delivery_address TEXT,
  payment_method TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_orders_user_id ON orders(user_id);
CREATE INDEX IF NOT EXISTS idx_orders_created_at ON orders(created_at DESC);

ALTER TABLE orders ENABLE ROW LEVEL SECURITY;

CREATE POLICY "orders_select" ON orders
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "orders_insert" ON orders
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "orders_update" ON orders
  FOR UPDATE USING (auth.uid() = user_id);
```

### Шаг 4: Перезапустите приложение
После выполнения SQL скриптов:
1. Закройте приложение в эмуляторе
2. Запустите заново: `flutter run`
3. Попробуйте оформить заказ

## ✅ Готово!
Теперь заказы должны работать без ошибок!
